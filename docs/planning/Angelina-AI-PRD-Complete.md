# 1. Executive Summary

## 1.1 Product Vision

ANGELINA represents a paradigm shift in personal productivity technology—an end-to-end, fully autonomous AI-powered operating system that transforms how solo founders, operators, and knowledge workers manage their digital work life. Unlike traditional assistants that merely respond to queries, ANGELINA fundamentally reimagines personal operations by deploying sophisticated AI agents that execute tasks across multiple tools, maintain persistent memory, and learn from every interaction.

The platform operates as a unified command center, enabling users to orchestrate their entire digital ecosystem—email, tasks, calendars, documents, CRM, and automation workflows—through natural voice conversations. ANGELINA handles the complete operational lifecycle: from intelligent email management, through AI-powered task execution, automated follow-ups, to comprehensive analytics and proactive suggestions.

## 1.2 Product Name Rationale

ANGELINA embodies the vision of a guardian intelligence—a name that conveys warmth, capability, and trust. Like a highly capable executive assistant or chief of staff, Angelina represents the intelligent connection point between the user and their entire digital ecosystem, handling complexity so the user can focus on high-value work.

## 1.3 Problem Statement

Modern knowledge workers face critical operational challenges:

| Problem | Impact |
|---------|--------|
| **Tool Fragmentation** | Average worker uses 13+ apps daily, losing 32 days/year to context switching |
| **Cognitive Overload** | 2.5 hours/day spent on email management alone |
| **Follow-up Fatigue** | 44% of sales lost due to lack of follow-up |
| **Memory Gaps** | Critical information scattered across tools, conversations lost |
| **Execution Gap** | Ideas and intentions don't translate to completed actions |

## 1.4 Solution

Angelina provides a unified, voice-first interface that:
- **Executes** tasks across all connected tools (not just suggests)
- **Remembers** everything—preferences, context, history
- **Orchestrates** complex multi-step workflows automatically
- **Learns** from every interaction to improve over time
- **Surfaces** insights proactively before you ask

## 1.5 Key Differentiators

| Feature | Traditional Assistants | ANGELINA |
|---------|----------------------|----------|
| Execution | Suggests actions | Completes actions autonomously |
| Memory | Session-only | Persistent long-term memory |

## 5.3 Data Flow Architecture

### Voice Command Flow

```
User Speaks → Deepgram STT (300ms) → Intent Recognition → 
LLM Processing → Tool Selection → n8n Workflow Trigger → 
Tool Execution → Response Generation → TTS (400ms) → User Hears
```

**Total Latency Target: <2 seconds end-to-end**

### Memory System Flow

```
User Input → Embedding Generation (ada-002) → 
Similarity Search (Pinecone) → Context Retrieval → 
LLM with Context → Response → Memory Update (if needed)
```

## 6.3 Intelligent Memory System

### 6.3.1 Memory Architecture

| Layer | Storage | Retention | Use Case |
|-------|---------|-----------|----------|

| Long-term Memory | Pinecone + PostgreSQL | Permanent | Preferences, client info, history |

### 6.3.2 Memory Operations

| Operation | Command Example | Action |
|-----------|-----------------|--------|
| **Save** | "Remember that Ramesh prefers WhatsApp" | Store as preference |
| **Recall** | "What's Ramesh's preferred communication?" | Vector search + retrieve |
| **Forget** | "Forget the old pricing" | Mark as deprecated |
| **Update** | "Actually, Ramesh now prefers email" | Version and update |
| **Summarize** | "What did we discuss this week?" | Aggregate and summarize |
| **Learn** | Implicit from interactions | Pattern recognition |

### 6.3.3 Memory Data Model

```
Memory Item:
- id: UUID
- user_id: UUID
- type: enum (preference, fact, interaction, document)
- content: text
- embedding: vector[1536]
- metadata: jsonb
  - entity: string (person, company, project)
  - confidence: float
  - source: string (explicit, inferred)
  - last_accessed: timestamp
- created_at: timestamp
- updated_at: timestamp
- deprecated_at: timestamp (nullable)
```

# 7. AI Agent Architecture

## 7.1 Agent Framework

ANGELINA deploys specialized AI agents, each optimized for specific domains:

| Agent | Function | Technologies | Trigger |
|-------|----------|--------------|---------|
| **Intent Router** | Classify user intent, route to appropriate agent | LLM + Few-shot | Every message |
| **Email Agent** | All email operations | Gmail API + LLM | "email", "send", "inbox" |
| **Task Agent** | Task management | ClickUp API + LLM | "task", "todo", "move" |
| **Calendar Agent** | Scheduling operations | GCal API + LLM | "schedule", "meeting", "calendar" |
| **Memory Agent** | Remember, recall, forget | Pinecone + LLM | "remember", "what did", "forget" |
| **Document Agent** | Generate files | Puppeteer + Templates | "create", "quotation", "proposal" |
| **Search Agent** | Information retrieval | Web Search + LLM | "search", "find", "look up" |
| **Analytics Agent** | Cost and usage queries | PostgreSQL + LLM | "cost", "usage", "how much" |

## 7.2 Agent Communication Protocol

```python
AgentRequest:
  - request_id: UUID
  - user_id: UUID
  - intent: string
  - entities: dict
  - context: list[Message]
  - memory_context: list[MemoryItem]
  - constraints: dict (budget, permissions)

AgentResponse:
  - request_id: UUID
  - status: success | partial | failed
  - result: any
  - actions_taken: list[Action]
  - cost: CostBreakdown
  - follow_up_required: bool
  - suggested_actions: list[string]
```
